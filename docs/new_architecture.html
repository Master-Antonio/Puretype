2 — ClearType on vs off: output equivalence

ClearType already on
GDI uses user's CT tuning (gamma, contrast from registry)
capture: R ≠ G ≠ B → full subpixel remap
ClearType off (grayscale or none)
ForceSubpixelRender: clone + CLEARTYPE_QUALITYGDI applies system CT registry settings to clone
capture: R ≠ G ≠ B → full subpixel remap

CreateFontIndirectW with CLEARTYPE_QUALITY respects system ClearType tuning (gamma, pixel geometry, contrast).
Both paths produce visually equivalent OLED-correct output.

3 — chromaKeep: subpixel chroma fraction preserved after tone mapping (RWBG)

v0.0.3 (old) v0.0.4 (fixed)

v0.0.3 values (18–50%) were ported from an LCD tone-mapper. On OLED, per-channel colour is the rendering mechanism — discarding it neutralises the subpixel benefit.

4 — WOLED FIR subpixel filter: 7-tap → 3-tap

v0.0.3 — 7-tap, span 1.75 physical px

weights (1+2+3+4+3+2+1)/16 — excessive blur → colour fringing on thin strokes

v0.0.4 — 3-tap, span 0.75 physical px

weights (0.25+0.50+0.25) — sufficient smoothing, no inter-pixel crosstalk
